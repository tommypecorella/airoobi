# INF-002 — ARIA DEV AGENT SETUP
**AIROOBI · Stage 0** · Last updated: 08 Mar 2026

## COS'È ARIA

ARIA è il dev agent AI di AIROOBI. Gira su un Raspberry Pi 5 sempre acceso (`skeeberrypi`), controllabile via Telegram. Usa OpenClaw come runtime e Claude Code (Claude Sonnet 4.6) come LLM. È il "dipendente tecnico" principale di AIROOBI in Stage 0 e Stage 1.

## CONFIGURAZIONE

### API Key Anthropic
```
Posizione: ~/.openclaw/agents/main/agent/auth-profiles.json
Metodo update: nano diretto (openclaw config set è inaffidabile in v2026.3.2)
Dopo modifica: systemctl --user restart openclaw
```

### Claude Code API Key
```
Metodo: export ANTHROPIC_API_KEY="sk-ant-..."
Persistenza: aggiunta in ~/.bashrc
Nota: -g flag NON supportato in Claude Code 2.1.63
Invocazione non-interattiva: claude -p "[prompt]"
```

## LEZIONI APPRESE (Stage 0)

- `openclaw config set` e `openclaw configure` sono inaffidabili per aggiornare la API key — usare sempre `nano` diretto
- Claude Code 2.1.63 non supporta il flag `-g` per la configurazione globale
- Il frontend standalone in un singolo file HTML è la scelta giusta per Stage 0: ARIA può modificarlo senza problemi di build
- Cache hit rate alto = buon segno di sessioni efficienti con ARIA
- Il Pi tende ad avere conflitti git se si lavora anche da Claude chat — fare sempre `git fetch + reset` prima di operazioni critiche
