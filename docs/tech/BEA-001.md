# BEA-001 — ARCHITETTURA BACKEND
**AIROOBI · Stage 0** · Last updated: 08 Mar 2026

---

## OVERVIEW

AIROOBI adotta un'architettura **serverless / BaaS** per Stage 0 e Stage 1, con Supabase come backend principale e Vercel come layer di deploy. Nessun server custom in questa fase — tutta la logica applicativa vive tra Supabase (PostgreSQL + Auth + RLS) e il frontend HTML/JS.

## SUPABASE — SCHEMA DB

### Tabella: `users` (gestita da Supabase Auth)
Estesa con profilo tramite trigger o tabella `profiles`.

### Tabella: `profiles`
```sql
id          uuid  (FK → auth.users.id)
username    text
lang        text  default 'it'
referral_code text unique
referred_by text
aria_balance integer default 0
streak_days  integer default 0
last_checkin date
created_at  timestamptz
```

### Tabella: `events`
```sql
id          uuid  primary key
user_id     uuid  (FK → auth.users.id)
type        text  -- 'checkin' | 'video' | 'referral' | 'airdrop_join'
aria_earned integer default 0
metadata    jsonb
created_at  timestamptz
```

### Tabella: `nft_rewards` *(da eseguire)*
```sql
id          uuid  default gen_random_uuid() primary key
user_id     uuid  (FK → auth.users.id)
nft_type    text  -- 'NFT_EARN' | 'NFT_ALPHA_BRAVE'
name        text
phase       text  default 'alpha'
source      text
created_at  timestamptz
```

### Tabella: `referrals`
```sql
id            uuid  primary key
referrer_id   uuid
referred_id   uuid
created_at    timestamptz
```

## ROW LEVEL SECURITY (RLS)

Tutte le tabelle hanno RLS abilitato. Pattern standard:

```sql
-- Lettura: solo il proprio record
create policy "Users see own data"
  on profiles for select
  using (auth.uid() = id);

-- Inserimento: chiunque autenticato
create policy "Auth users can insert"
  on events for insert
  with check (auth.uid() = user_id);
```

## AUTH FLOW

```
User → Form signup → Supabase Auth → Email confirm (Postmark)
                                   → JWT token → localStorage
User → Form login  → Supabase Auth → JWT refresh → dashboard
```

- Conferma email obbligatoria
- JWT scadenza: 1h (refresh automatico)
- SMTP: Postmark (server AIROOBI, ID 18417614) — **in attesa approvazione**
