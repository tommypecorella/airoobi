**M_atto2_02 · Total_blocks algorithm derivation:**
```sql
CREATE OR REPLACE FUNCTION airdrop_total_blocks_derive(
  p_object_value_eur NUMERIC,
  p_block_price_aria INTEGER DEFAULT 15
) RETURNS INTEGER LANGUAGE SQL IMMUTABLE AS $$
  SELECT CEIL(p_object_value_eur * 2 / (p_block_price_aria * 0.10))::INT;
  -- object_value × 2 / block_price_eur = total_blocks for 2× revenue
  -- 15 ARIA × 0.10 = €1.50 per block · 700 EUR object → 934 blocks
$$;

GRANT EXECUTE ON FUNCTION airdrop_total_blocks_derive(NUMERIC, INTEGER) TO authenticated;
```

**M_atto3_01 · `compute_checkmate_blocks` RPC (Scacco Matto logic):**
```sql
CREATE OR REPLACE FUNCTION compute_checkmate_blocks(
  p_user_id UUID,
  p_airdrop_id UUID
) RETURNS JSONB LANGUAGE plpgsql STABLE AS $$
DECLARE
  v_user_score NUMERIC;
  v_user_blocks INTEGER;
  v_leader_score NUMERIC;
  v_leader_blocks INTEGER;
  v_leader_user_id UUID;
  v_blocks_remaining INTEGER;
  v_total_blocks INTEGER;
  v_blocks_sold INTEGER;
  v_sold_pct NUMERIC;
  v_scacco_matto_threshold_pct NUMERIC := 85.0;
  v_blocks_to_overtake INTEGER;
  v_blocks_to_checkmate INTEGER;
  v_leader_blocks_to_self_checkmate INTEGER;
  v_user_multiplier NUMERIC;
  v_leader_multiplier NUMERIC;
BEGIN
  -- Get airdrop state
  SELECT total_blocks, blocks_sold, (blocks_sold * 100.0 / total_blocks)
  INTO v_total_blocks, v_blocks_sold, v_sold_pct
  FROM airdrops WHERE id = p_airdrop_id;

  v_blocks_remaining := v_total_blocks - v_blocks_sold;

  -- Get current scoreboard top (via calculate_winner_score RPC W3 deployed)
  -- Extract leader + user scores
  -- ... [implementazione query dettagliata]

  -- Math: per overtake leader corrente
  -- blocks_needed = (leader_score / user_multiplier)^2 - user_blocks_current
  v_blocks_to_overtake := CEIL(POWER(v_leader_score / v_user_multiplier, 2)) - v_user_blocks;

  -- Math: per scacco matto al campo (anche se top2 prende tutti remaining)
  -- consider max possible top2 score = sqrt(top2_blocks + remaining) × top2_multiplier
  v_blocks_to_checkmate := CEIL(POWER((SQRT(v_leader_blocks + v_blocks_remaining) * v_leader_multiplier) / v_user_multiplier, 2)) - v_user_blocks;

  -- Math: leader self-checkmate
  v_leader_blocks_to_self_checkmate := ...; -- analogous

  RETURN jsonb_build_object(
    'user_score_current', v_user_score,
    'user_blocks_current', v_user_blocks,
    'leader_user_id', v_leader_user_id,
    'leader_score_current', v_leader_score,
    'leader_blocks_current', v_leader_blocks,
    'blocks_to_overtake_leader', GREATEST(v_blocks_to_overtake, 0),
    'aria_cost_to_overtake', GREATEST(v_blocks_to_overtake, 0) * 15,
    'blocks_to_checkmate_field', GREATEST(v_blocks_to_checkmate, 0),
    'aria_cost_to_checkmate', GREATEST(v_blocks_to_checkmate, 0) * 15,
    'leader_blocks_to_self_checkmate', GREATEST(v_leader_blocks_to_self_checkmate, 0),
    'scacco_matto_threshold_sold_pct', v_scacco_matto_threshold_pct,
    'current_sold_pct', v_sold_pct,
    'scacco_matto_active', v_sold_pct >= v_scacco_matto_threshold_pct
  );
END;
$$ SECURITY DEFINER;

GRANT EXECUTE ON FUNCTION compute_checkmate_blocks(UUID, UUID) TO authenticated;
```

### RPCs

**`detect_airdrop_end_event(p_airdrop_id UUID)`** · cron-callable:
- Check 3 trigger conditions:
  - DEADLINE: `NOW() >= deadline`
  - SOLD-OUT: `blocks_sold >= total_blocks`
  - SCACCO MATTO: `compute_checkmate_blocks(leader_user_id, airdrop_id)['scacco_matto_active'] = TRUE AND sold_pct >= 85`
- If any triggered AND status IN ('presale', 'sale'):
  - Update status → 'waiting_seller_acknowledge'
  - Compute winner candidate (snapshot via `calculate_winner_score` W3 deployed)
  - Set `winner_candidate_user_id`
  - Set `end_event_triggered_at = NOW()`
  - Set `seller_acknowledge_sla_deadline = NOW() + INTERVAL '24 hours'`
  - Set `end_event_trigger_type` accordingly
  - Trigger push T1 seller via edge function `notify-seller-acknowledge`
  - Trigger broadcast push T1 buyers attivi "Evento terminato · in attesa conferma seller · {time} per finalizzazione"

**`seller_acknowledge_airdrop(p_airdrop_id UUID, p_decision TEXT)`** · seller-only:
- Validation: `auth.uid() = airdrops.seller_id`
- Validation: status = 'waiting_seller_acknowledge'
- Validation: `NOW() < seller_acknowledge_sla_deadline`
- Validation: `p_decision IN ('accept', 'annulla')`
- Update: `seller_acknowledge_decision = p_decision`, `seller_acknowledge_decided_at = NOW()`
- If accept → call `execute_draw(p_airdrop_id)` (W3 deployed · winner ufficial + payment + NFT)
- If annulla → call `refund_airdrop(p_airdrop_id)` (W3 deployed · refund all + top-3 NFT consolation)
- Trigger push T1 multi-channel winner announcement (post-acceptance · NEVER before)

**Cron `auto_accept_silent_seller`** · run every 5 minutes:
- Query airdrops WHERE status='waiting_seller_acknowledge' AND seller_acknowledge_sla_deadline < NOW()
- Per each: call `seller_acknowledge_airdrop(id, 'accept')` con simulated seller (SECURITY DEFINER bypass auth check + log `auto_accept_silent`)
- Update `seller_acknowledge_decision = 'auto_accept_silent'`
- Trigger push T1 seller "Auto-accept executed post-24h SLA · airdrop finalized"

### RPCs

**`claim_airdrop_prize(p_airdrop_id UUID, p_shipping_address JSONB, p_shipping_phone TEXT, p_shipping_notes TEXT)`**
- Validation: airdrops.winner_id = auth.uid() · status='completed'
- Insert airdrop_claims row
- Trigger push T1 seller "Vincitore ha richiesto consegna · indirizzo allegato"

**`confirm_airdrop_dispatched(p_airdrop_id UUID, p_tracking_number TEXT, p_courier TEXT)`**
- Validation: airdrops.seller_id = auth.uid() · claim exists · status='completed'
- Update airdrop_claims: dispatched_at, tracking_number, courier
- Trigger push T1 winner "Spedito · tracking #{number} con {courier}"

**`confirm_airdrop_received(p_airdrop_id UUID, p_rating INTEGER, p_rating_notes TEXT)`**
- Validation: winner_id = auth.uid() · dispatched_at NOT NULL
- Update airdrop_claims: received_at, rating, rating_notes
- Trigger push T1 seller "Consegna confermata · rating: {N}/5"
- Update seller reputation score (computed aggregate)

**`open_airdrop_dispute(p_airdrop_id UUID, p_reason TEXT, p_evidence_urls JSONB)`**
- Validation: auth.uid() IN (winner_id, seller_id) · claim exists · within 30gg dispute window
- Insert airdrop_disputes row
- Trigger push T1 admin "Dispute opened {airdrop_id} · review needed"
- Trigger push T1 altra parte "Dispute aperta dal seller/winner · admin sta esaminando"

**`resolve_airdrop_dispute(p_dispute_id UUID, p_resolution TEXT, p_refund_seller_pct NUMERIC, p_refund_buyer_pct NUMERIC, p_resolution_notes TEXT)`** · admin-only:
- Validation: is_admin() · dispute status='pending_review'
- Update airdrop_disputes: resolution_*, resolved_at
- Execute refund actions per pct
- Trigger push T1 both parties "Dispute resolved · vedi dettagli /venditore o /profilo"
- Update reputation scores

### Cron jobs

**`cron_check_dispatch_timeout`** · daily:
- Query claims WHERE dispatched_at IS NULL AND claimed_at < NOW() - INTERVAL '14 days'
- For each: auto-create dispute (opener=winner · reason='dispatch_timeout_14d')

**`cron_check_dispute_window_close`** · daily:
- Query claims WHERE received_at IS NOT NULL AND received_at < NOW() - INTERVAL '30 days' AND NO dispute exists
- For each: mark airdrop_claims as `finalized` (no further dispute possible)
