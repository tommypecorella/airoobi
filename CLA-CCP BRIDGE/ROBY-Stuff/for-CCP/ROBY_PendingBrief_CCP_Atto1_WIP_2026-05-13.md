### Area 1 · EVALOBI table + lifecycle

**Migration M1:**
```sql
CREATE TABLE evalobi (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  token_id BIGINT GENERATED BY DEFAULT AS IDENTITY UNIQUE,  -- on-chain compatible
  current_owner_id UUID NOT NULL REFERENCES profiles(id),
  original_seller_id UUID NOT NULL REFERENCES profiles(id),

  -- Object metadata (Livello B)
  object_title TEXT NOT NULL,
  object_brand TEXT NOT NULL,
  object_model TEXT NOT NULL,
  object_condition TEXT CHECK (object_condition IN ('nuovo','come_nuovo','usato_eccellente')),
  object_year INTEGER,
  object_category TEXT NOT NULL,
  object_photo_hashes JSONB,  -- foto IPFS/storage hashes

  -- Evaluation outcome
  evaluation_outcome TEXT CHECK (evaluation_outcome IN ('GO','NO_GO','NEEDS_REVIEW')),
  evaluation_price_range JSONB,  -- {min_eur, max_eur, suggested_eur}
  evaluation_reasoning TEXT,
  evaluation_admin_id UUID REFERENCES profiles(id),
  evaluated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

  -- Versioning (re-evaluation support)
  version INTEGER NOT NULL DEFAULT 1,
  supersedes_evalobi_id UUID REFERENCES evalobi(id),  -- old version pointer

  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_evalobi_owner ON evalobi(current_owner_id);
CREATE INDEX idx_evalobi_original_seller ON evalobi(original_seller_id);
CREATE INDEX idx_evalobi_outcome ON evalobi(evaluation_outcome);

-- Storico append-only per re-evaluation chain
CREATE TABLE evalobi_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  evalobi_id UUID NOT NULL REFERENCES evalobi(id),
  event_type TEXT CHECK (event_type IN ('minted','transferred','re_evaluated','superseded')),
  event_metadata JSONB,
  prev_owner_id UUID REFERENCES profiles(id),
  new_owner_id UUID REFERENCES profiles(id),
  triggered_by UUID REFERENCES profiles(id),
  occurred_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**RPC M2:**
- `mint_evalobi(p_evaluation_request_id, p_outcome, p_price_range, p_reasoning, p_admin_id)` → emette EVALOBI per seller
- `transfer_evalobi(p_evalobi_id, p_new_owner_id)` → seller può trasferire EVALOBI a altro user wallet
- `re_evaluate_evalobi(p_evalobi_id)` → trigger valutazione nuova · paga 200 ARIA · nuova EVALOBI v2 · vecchia marked `superseded_by`

**RLS:**

### Area 3 · Swap functionality (tokens conversion)

**Migration M5:**
```sql
CREATE TABLE token_swaps (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id),
  from_token TEXT CHECK (from_token IN ('ARIA','KAS','ROBI','EUR')),
  to_token TEXT CHECK (to_token IN ('ARIA','KAS')),  -- ROBI receive-only
  from_amount NUMERIC(20,8) NOT NULL,
  to_amount NUMERIC(20,8) NOT NULL,
  exchange_rate NUMERIC(20,8) NOT NULL,  -- rate at conversion time
  fee_aria NUMERIC(20,8) DEFAULT 0,
  status TEXT CHECK (status IN ('pending','completed','failed')) DEFAULT 'pending',
  external_tx_hash TEXT,  -- Kaspa tx hash per swap KAS-side · NULL altrimenti
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

CREATE INDEX idx_swaps_user ON token_swaps(user_id, created_at DESC);
```

**RPC M6:**
- `swap_kas_to_aria(p_user_id, p_kas_amount)` · valida KAS balance · debit KAS · credit ARIA · log
- `swap_aria_to_kas(p_user_id, p_aria_amount)` · valida ARIA balance · debit ARIA · credit KAS · log · oracle KAS_EUR rate
- `swap_robi_to_aria(p_user_id, p_robi_amount)` · valida ROBI balance · debit ROBI · credit ARIA · log · rate da Treasury formula `treasury_balance_eur / robi_circulating` (ROBI price dinamico)
- `swap_robi_to_kas(p_user_id, p_robi_amount)` · valida ROBI balance · debit ROBI · credit KAS · log
- `buy_aria_eur(p_user_id, p_eur_amount)` · Stripe checkout · post-success · credit ARIA at €0.10 fisso

**Guard rules:**
- ❌ `swap_aria_to_robi` · NON ESISTE come RPC
- ❌ `swap_kas_to_robi` · NON ESISTE come RPC
- Validate input range · slippage protection KAS↔ARIA · oracle freshness check

- `/wallet` o `/swap` page · form swap intuitivo · 3 conversion paths visibili (KAS↔ARIA + ROBI→ARIA/KAS)

### Area 7 · Reattività Dashboard pubblico + notification email constraint

**Strategic rationale (Skeezu 13 May 2026 evening):** SLA 48h promise deve essere supportata da transparency-as-a-feature · dashboard pubblico mostra response time reale rispetto promise · brand trust + pre-emptive objection killer per seller hesitanti.

**Migration M11:**
```sql
-- Materialized view per metriche aggregate (refresh ogni 5 min via pg_cron)
CREATE MATERIALIZED VIEW sla_metrics_30d AS
SELECT
  COUNT(*) AS total_requests_30d,
  AVG(EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600) AS avg_response_hours,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600) AS median_response_hours,
  (COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 <= 48)) * 100.0 / COUNT(*) AS pct_under_48h,
  (COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 <= 24)) * 100.0 / COUNT(*) AS pct_under_24h,
  -- Distribution buckets
  COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 <= 12) AS bucket_0_12h,
  COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 > 12 AND EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 <= 24) AS bucket_12_24h,
  COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 > 24 AND EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 <= 36) AS bucket_24_36h,
  COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 > 36 AND EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 <= 48) AS bucket_36_48h,
  COUNT(*) FILTER (WHERE EXTRACT(EPOCH FROM (decided_at - created_at)) / 3600 > 48) AS bucket_over_48h,
  NOW() AS refreshed_at
FROM evaluation_requests
WHERE decided_at IS NOT NULL
  AND created_at >= NOW() - INTERVAL '30 days';

CREATE UNIQUE INDEX idx_sla_metrics_30d ON sla_metrics_30d(refreshed_at);

-- Refresh schedule
SELECT cron.schedule('refresh-sla-metrics-30d', '*/5 * * * *', 'REFRESH MATERIALIZED VIEW CONCURRENTLY sla_metrics_30d');
```
